A Request Method is safe if the client does not expect any state change in the origin server. Safe Methods don't make any changes in the origin itself.

GET

Retrieves information identified by the Request-URI in a form of an entity.

The primary way of retrieving information and resources. It ‘request transfer of a current selected representation for a target resource’.

Syntax:GET /example/index.html

HEAD

It is the same as the GET method except that the server must not return the payload in the response. Instead, it only return the headers much like if it had been a Get method.

In other words, it returns the metadata of the response.

Syntax:HEAD /example/index.html

OPTIONS

Requests information about the communications options available for the target resource but does not specify a resource action.

Syntax:OPTIONS /example/index.html HTTP/1.1

OPTIONS * HTTP/1.1(refers to the entire server)

TRACE

Requests a remote, application-level loop-back test of the request message along the path to the target resource.

Syntax:TRACE http://www.example.com HTTP/1.1
